# Objetivo da Avaliação

Esta atividade tem como objetivo validar os conhecimentos adquiridos durante as aulas de Vue.js.

O aluno deverá demonstrar sua capacidade de:

- Compreender requisitos de software;
- Interpretar wireframes;
- Desenvolver componentes reutilizáveis;
- Utilizar Vue Router;
- Aplicar conceitos de reatividade;
- Estruturar uma SPA (Single Page Application).

# 1. Funcionalidade - 40%

## O que será avaliado

Funcionamento geral da aplicação.

### Pontos avaliados

- Navegação entre páginas
- Cadastro de avaliações
- Cálculo da média
- Exibição do ranking
- Exibição dos detalhes
- Atualização automática dos dados

### Nota Máxima

40 pontos

---

# 2. Conceitos Vue.js - 30%

## O que será avaliado

Uso correto dos conceitos vistos em sala.

### Conceitos esperados

#### Templates

- Interpolação
- v-if
- v-else
- v-for
- v-bind
- v-on

#### Componentes

- Criação de componentes reutilizáveis
- Uso de Props

#### Reatividade

- ref()
- Atualização automática da interface

#### Rotas

- Vue Router
- Navegação entre páginas

### Nota Máxima

30 pontos

---

# 3. Organização do Projeto - 15%

## O que será avaliado

Estrutura e legibilidade do código.

### Pontos avaliados

- Separação em componentes
- Organização das pastas
- Nomes significativos
- Código limpo
- Reutilização de componentes

### Nota Máxima

15 pontos

---

# 4. Interface e Experiência - 10%

## O que será avaliado

Qualidade visual da aplicação.

### Pontos avaliados

- Layout organizado
- Alinhamento dos elementos
- Espaçamento adequado
- Responsividade básica
- Consistência visual

### Observação

Não é necessário criar um design avançado.

O foco é a organização da interface.

### Nota Máxima

10 pontos

---

# 5. Documentação - 5%

## O que será avaliado

Documentação entregue junto ao projeto.

### README obrigatório

O README deverá conter:

- Nome do projeto
- Nome do aluno
- Como executar o projeto
- Tecnologias utilizadas
- Prints das telas

### Nota Máxima

5 pontos

# Itens que Descontam Pontos

Os itens abaixo poderão reduzir a nota final:

| Problema | Impacto |
|-----------|----------|
| Aplicação não inicia | Grave |
| Rotas não funcionam | Grave |
| Erros no console | Médio |
| Componentes duplicados sem necessidade | Médio |
| Código desorganizado | Médio |
| Ausência de README | Leve |
| Layout quebrado | Leve |

---

# Critérios de Excelência

Os alunos poderão se destacar ao implementar funcionalidades extras como:

- Filtros de cafés
- Busca por nome
- Persistência com LocalStorage
- Dark Mode
- Ordenação dinâmica
- Melhorias visuais
- Responsividade avançada

Esses itens não são obrigatórios, mas demonstram domínio adicional da tecnologia.
