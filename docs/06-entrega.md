# O que deve ser entregue

Cada aluno deverá entregar:

## Opção 1 (Recomendada)

Link do repositório GitHub ou GitLab contendo:

- Código-fonte completo
- README preenchido
- Projeto funcionando

---

## Opção 2

Arquivo compactado (.zip)

Contendo:

- Código-fonte completo
- README preenchido

⚠️ Não incluir a pasta `node_modules`.

---

# Estrutura Esperada

Exemplo:

```txt
coffee-quality-challenge/
│
├── src/
│   ├── assets/
│   ├── components/
│   ├── router/
│   ├── views/
│   ├── layout/
│   └── App.vue
│
├── public/
│
├── package.json
├── vite.config.js

```

# README Obrigatório

O projeto deverá conter um arquivo:

```txt
README.md
```

na raiz do repositório.

---

## O README deve conter

### Identificação

- Nome do aluno
- Turma
- Disciplina

---

### Sobre o Projeto

Breve descrição da aplicação desenvolvida.

Exemplo:

> Sistema para avaliação sensorial de cafés inspirado na metodologia SCA.

---

### Tecnologias Utilizadas

Exemplo:

- Vue 3
- Vue Router
- Vite
- JavaScript
- CSS

---

### Como Executar o Projeto

Exemplo:

```bash
npm install
npm run dev
```

### Funcionalidades Implementadas

Exemplo:

- Home

- Ranking
- Detalhes do Café

# Funcionalidades Obrigatórias

A aplicação deverá possuir:

## Home

- Dashboard com indicadores

## Avaliações

- Cadastro de avaliações
- Cálculo automático da média

## Ranking

- Tabela ordenada por nota

## Detalhes

- Visualização completa de uma avaliação

## Rotas

- Navegação entre páginas

## Componentes

- Pelo menos 3 componentes reutilizáveis

## Página 404

- Tratamento para rotas inexistentes
